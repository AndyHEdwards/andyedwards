Curriculum Vitae

I am a web developer that has an affinity to the front end and a keen eye for detail. I love programming because of the combination of challenge and creativity that it offers.

The area that makes me most excited at the moment is the rapid development of font end Javascript technologies, in particular React.js and functional reactive programming. I have a background in hospitality management where I have developed excellent communication skills and confidence in working closely with people. I took a shining to programming because its the perfect way to be able to push my abilities, express myself, and at the same time constantly be challenged.

After attending an 18 week dev bootcamp I am ready to hit the ground running in a new career. Currently I am working on re-writing “Book It”, the app that we made for our final project, from Angular.js to React.js.

Relevant Skills

  • Experience with technology - Rails, Rspec, Javascript, Angular.js, React.js, Git, Object Orientated and Functional programming, Document and Relational databases, ORMs, Consuming API’s, SQL, HTML, CSS, Jquery and Node (Express, Mocha, Chai, Webpack, JSPM, Immutable.js and many more Node packages).
  • Agile methodologies - Pair programming, Stand-ups, Kanban boards, retrospectives and sprints.
  • Team focus - I am at my best when working as part of a team, and teams feed off my positive, can-do attitude. I am also well versed in check-ins and providing feedback when working with others.
  • Good communication - I have excellent communication skills and can easily and clearly express my thoughts and work to others. A big part of communication is being able to listen as well as talk, so I practice mindful listening to try and truly understand what people are saying.
  • Attention to detail - I take pride in all work that I do, and have very high standards that I won’t compromise on. I always apply industry best practices, to give clarity to myself and others that may someday have to work on my code.
  • Self motivated - Now that I have been re-immersed into a learning environment I have found something that I thought I lost. A genuine excitement and passion to learn, which I wish to carry on in my career in web development.

Relevant Experience

Enspiral Dev Academy
Enspiral Dev Academy is an 18 week intensive bootcamp that aims to produce junior developers that are ready for the tech industry. EDA focuses on two core sets of skills, hard tech and soft interpersonal skills. The first 9 weeks (‘phase 0’) are learning the fundamentals of programming remotely and the following 9 weeks (‘bootcamp’) are on site, Monday to Friday, learning in a realistic industry environment.

I had the opportunity to re-do phases one and two which gave me a chance to really solidify my understanding of what I’ve learnt. More importantly it gave me a chance to act as more of a mentor to the others in my group which was a really awesome learning experience.

Phase 0

Was about 30 hours a week of remotely learning the fundamentals of programming.This is where I was introduced to the basics such as HTML, CSS, Javascript and Ruby. It involved completing weekly challenges and making our solutions available for coaches to review through Github. One of the most important lessons I learnt from phase 0 was how to manage my time effectively. I was still working a full time job, and learning to juggle the two was a really useful experience.

Phase 1 - Rails

The first three weeks of the course were spent learning Ruby on Rails, which was my introduction to programming with a framework. It was awesome to see how quickly and painlessly it can be to get an app up and running. There were a lot of firsts during this phase including using different types of databases and ORMs, testing with Rspec and learning object orientated programming.

During this phase we were taught effective protocols for working in a team environment, for example how to give actionable feedback and to how to notice when we and our team mates are at our limit and what to do in those scenarios.

Phase 2 - Javascript

At this point we started to really get stuck into javascript. This is where I learnt fundamental js concepts such as callbacks, promises, closures and AJAX requests. Another pivotal moment in phase 2 was when I learnt to produce and consume APIs and got my head around how asynchronous coding works.

This phase was a transitional stage where we started to be able to link together everything we learnt and start to make full stack applications, and got an understanding the concept of service orientated architecture.

Phase 3 - Junior Developers

The last phase of the course was when we started to be treated as junior developers and the coaches took on more of a mentoring senior dev role. We were given a lot more room to dive into our own self guided learning which is where I discovered my passion for front end javascript frameworks. React grabbed my attention and I quickly started to get stuck into functional programming and coupling React with Redux to make more fleshed out apps.

The projects that we were making at this stage started to become more involved so one of the main focuses to keep them working was to have a very tight git flow and constant communication within our team. The last week was spent making our final project. We made an app called “Book it”, which was a restaurant table booking app designed to remove the archaic way tables are currently booked by using phone, paper and pen. The architecture that we chose was a rails backend that used a postgresql database and a seperate front-end app that was built with Angular. The two communicated with an API that we made.

Hospitality July 2013 - September 2015

From working in hospitality for just over three years I have acquired great communication skills that will be of value to any company. Whether it is being able to successfully communicate to potential clients, or just keeping up communication when working in a team environment. While working in Wellington I would be dealing with potential function clients on an almost daily basis, and often be involved with their events from planning all the way through to execution. This involved constantly keeping in contact, while also managing their expectations for what could or couldn’t be done.

At the age of 20 I was made a manager so I have experience in leading a team in a high intensity, stressful environment, which is a skill that I’m sure will come into its own when deadlines are looming. It also gave me a greater understanding of how business’s are run behind the scenes.

Rolling Stone

Rolling Stone

Project Description

This app was a project that I was working on during bootcamp. The initial idea was to make a travel journal/blogging app. Due to time constraints and an increase in interest in other areas I lost time to finish it, but learnt some really valuable lessons none the less.

The main area of learning was around building a login from the ground up without using a gem. I learnt a lot about encryption and security around user authentication.

One problem that I ran into during deployment was setting up an email server for produciton. I spent a few hours battling with cryptic heroku errors but eventually found out they were being caused by the app trying to send a confirmation email, but without an email client. However, once this was sorted the rest of the deployment was a breeze.

Cow Says Moo

Cow Says Moo

Project Description

This was a two day, group project, that we made when beginning to learn javascript in bootacmp. The goal was to make an app that was beautiful, funny or interesting, without any emphasis on real functionality. We made Cow Says Moo by using Javascript and the jQuery library.

During the entirety of this project we implemented agile methodoligies and all the programming was done in pairs. This was a really useful project for gaining greater understanding of basic javascript concepts and learning how to manipulate the dom using jQuery.

Angular
Book It

Book It

Project Description

This is the app that we made for our final project. The front end is an app that is made in Angular and served seperately from the backend which is made in Ruby. The front end is served off Github Pages and the backend is served from Heroku.

One of the bigger challanges of this project was succesfull communication between the front and back end in the sense of sercure user authentication. We did this using devise on the Ruby side and ng-token-auth on the Angular side.

We also made a REST API so the client side could retreive all of the current users booking information, and updates about the maerchants availabilities.
